## AI

The AI domain is Trustbird's first abstraction for providers, prompts and **suggestions**. AI output is never an automatic decision: every result is stored as a pending suggestion that requires human approval.

Provider-specific HTTP clients and credentials live in the application. This package stores the domain records, approval metadata and audit logs.

### Concepts

- **Provider** (`AiProviderDriver`): openai, anthropic, azure, custom — settings stay non-secret metadata
- **Prompt**: reusable prompt body with a stable `key`
- **Suggestion** (`AiSuggestionStatus`): pending, approved, rejected, withdrawn
- **Kind** (`AiSuggestionKind`): policy, risk, measure, evidence, general
- **Audit log**: `ai_suggestion_logs` for created/approved/rejected/withdrawn events
- **Driver contract** (`GeneratesAiSuggestions`): application-implemented generator; Trustbird only records the result

### Register a provider

```php
use Trustbird\Facades\Trustbird;
use Trustbird\Ai\Enums\AiProviderDriver;

$provider = Trustbird::ai()->registerProvider(
    name: 'Workspace OpenAI',
    driver: AiProviderDriver::OpenAi,
    settings: ['default_model' => 'gpt-4.1'],
);
```

### Create a prompt

```php
$prompt = Trustbird::ai()->createPrompt(
    key: 'control-from-interview',
    name: 'Control suggestion',
    body: 'Propose a plain-language control for: {{answer}}',
    purpose: 'measure',
);
```

### Record a suggestion

```php
use Trustbird\Ai\Enums\AiSuggestionKind;

$suggestion = Trustbird::ai()->suggest(
    output: ['text' => 'Require MFA for administrator accounts'],
    kind: AiSuggestionKind::Measure,
    provider: $provider,
    prompt: $prompt,
    title: 'Add MFA control',
);
```

### Approve or reject

```php
Trustbird::ai()->approve(
    suggestion: $suggestion,
    reviewedById: $person->id,
    reviewNotes: 'Fits our environment.',
);

Trustbird::ai()->reject(
    suggestion: $suggestion,
    reviewedById: $person->id,
    reviewNotes: 'Too broad.',
);
```
