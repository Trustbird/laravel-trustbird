# Contributing to Trustbird

Thank you for your interest in contributing to Trustbird.

Trustbird is an open-core compliance platform focused on helping organisations build and maintain an Information Security Management System (ISMS) through understandable workflows and automation.

## Before you start

Please discuss significant changes through a GitHub Issue before investing substantial development effort.

This helps ensure alignment with the project roadmap and architecture.

## Development principles

When contributing to Trustbird, please keep the following principles in mind:

- Simplicity over complexity.
- Laravel-first architecture.
- Clear and understandable user experiences.
- Security and auditability by design.
- Frameworks are outputs, not the primary user experience.
- Avoid unnecessary third-party dependencies.

## Pull Requests

Please ensure that:

- Code follows PSR-12 standards.
- New functionality includes tests where applicable.
- Existing tests continue to pass.
- Documentation is updated when behaviour changes.
- Breaking changes are clearly documented.
- `CHANGELOG.md` remains publishable (no `Unpublished` release headers; use ISO dates like `2026-07-07`).

## Reporting issues

If you encounter a bug or have a feature request, please open a GitHub Issue.

For security vulnerabilities, please follow the process described in SECURITY.md.

## Code of Conduct

By participating in this project, you agree to act respectfully and constructively towards other contributors.
